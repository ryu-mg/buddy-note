# buddy-note Ideas

앞으로 추가하고 싶은 제품 아이디어를 모아두는 문서.

이 문서는 확정된 스펙이 아니라 backlog 성격이다. 구현 전에 `AGENTS.md`,
`rules/architecture.md`, `rules/code-conventions.md`, `DESIGN.md`를 다시 보고
구체적인 스펙과 migration/API/UI 범위를 별도로 잠근다.

---

## 1. 유료 사용자 멤버십

유료 사용자에게만 열리는 기능을 추가한다.

목표:
- 무료 사용자와 유료 사용자의 제품 경험을 명확히 분리한다.
- buddy-note의 핵심 가치인 기억 누적, 공유 이미지, 테마, 가족 공유를 유료 전환 포인트로 만든다.

초기 후보:
- 프리미엄 테마
- 고급 공유 이미지 템플릿
- 월간/연간 하이라이트 앨범
- 더 긴 memory summary 보존
- 가족 알림/공유 기능

검토 필요:
- 결제 제공자
- 구독 상태 저장 방식
- RLS와 entitlement 체크 위치
- 무료 사용자에게 잠긴 기능을 어떻게 보여줄지

## 2. 무료 체험판

신규 사용자에게 일정 기간 동안 유료 기능을 체험할 수 있게 한다.

목표:
- 온보딩 직후 유료 기능의 가치를 먼저 느끼게 한다.
- 결제 전환 전 마찰을 낮춘다.

아이디어:
- 첫 가입 후 N일 무료 체험
- 첫 반려동물 등록 후 N일 무료 체험
- 첫 일기 생성 후 N일 무료 체험

검토 필요:
- N일 기준
- 체험 시작 트리거
- 체험 종료 전 안내 UI
- 체험 종료 후 잠금 처리
- 중복 체험 방지

## 3. 패밀리 기능

내가 오늘 일기를 작성하면 가족들에게도 푸시 알림이 가도록 한다.

목표:
- buddy-note를 개인 기록 앱에서 가족 공유 경험으로 확장한다.
- 보호자 여러 명이 같은 반려동물의 하루를 같이 따라가게 한다.

아이디어:
- 반려동물 단위 가족 초대
- 가족 멤버 권한: 보기 전용, 작성 가능, 관리자
- 새 일기 작성 시 가족에게 push notification
- 공개 프로필과 별개로 가족 전용 피드 제공

검토 필요:
- `pets.user_id` 단일 owner 구조를 어떻게 확장할지
- 가족 초대 링크/승인 플로우
- push token 저장과 알림 권한 요청
- 가족 멤버별 RLS 정책
- 알림 빈도 제한

## 4. 테마 수정

유료 사용자에게만 테마 UI 변경 기능을 제공한다.

목표:
- 각 강아지의 성격과 분위기에 맞는 앨범 느낌을 만들 수 있게 한다.
- 유료 멤버십의 가시적인 보상으로 사용한다.

아이디어:
- 기본 테라코타 테마는 유지
- 유료 전용 테마 preset 제공
- 공개 프로필 `/b/[slug]`와 공유 이미지에 테마 반영
- 계절/무드 기반 테마

검토 필요:
- DESIGN.md의 locked visual style과 충돌하지 않는 범위
- purple/violet 금지 등 브랜드 guardrail 유지
- 테마 토큰 저장 위치
- 테마가 public profile ISR/cache에 미치는 영향
- 무료 사용자에게 테마 preview만 보여줄지 여부
