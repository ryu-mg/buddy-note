UI/UX 개선 로드맵 v0.2

현재 buddy-note는 브랜드 정체성(폴라로이드·테라코타·명조체)은 단단하지만 화면에서 "강아지"라는 주인공이 안 보입니다. 이 문서는 DESIGN_TODOS.md에 정리된 15개 티켓을 화면으로 보여줍니다. 좌측 네비에서 각 티켓을 확인하세요.

근본 진단 4가지

01

캐릭터 엔진인데 캐릭터가 없음

제품은 "강아지의 성격을 1년 동안 기억"하는데, 홈/프로필 어디에도 주인공인 강아지의 존재감이 배치되어 있지 않다.

02

따뜻함이 기울기 1.2도에만 의존

"귀엽지만 sloppy하지 않음"이 이상이지만, UI 실행은 "정확함"으로 과하게 기움. 카피·카드 기울기에만 warmth가 고립.

03

톤 분열 (Diary ↔ UI)

Diary는 강아지 1인칭인데 UI 카피는 보호자 3인칭("사진 한 장을 올려주세요"). 대화하는 앱이 아닌 도구 앱 느낌.

04

감정 moment 부재

빈 상태·완료·마일스톤에 셀레브레이션이 없음. 재방문 동기가 약하고 moat(persona 기억)가 UI에 투명.

D-4 · Tier 1 · 1-2일

브랜드 SVG 일러스트 3개

buddy-happy

스펙
  • viewBox 96×96, 실제 사용 48-120px
  • 단일 stroke 2px, fill은 currentColor + opacity로 그림자만
  • React component (components/illustrations/*.tsx), 외부 SVG 파일 X
  • role="img" aria-label 필수. 장식용은 aria-hidden
  • 다크모드 자동 반전 (currentColor 상속)
