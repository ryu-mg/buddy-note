# First-Entry Tutorial Spec

## Summary

튜토리얼은 회원가입 후 강아지 등록을 마친 사용자가 앱에 최초 1회 진입했을 때 보여주는 짧은 기능 안내다. 목적은 마케팅 설명이 아니라 사용자가 첫 세션 안에서 앱의 기본 구조를 이해하고, 곧바로 첫 일기 작성까지 이동할 수 있게 하는 것이다.

이번 버전에서 설명할 핵심 기능은 세 가지다.

- 캘린더 뷰: 홈에서 날짜별 기록을 확인한다.
- 주간 뷰: 하단 네비게이션바 가장 좌측 탭(`/week`)에서 최근 흐름을 모아본다.
- AI 일기 생성: 사진과 메모를 남기면 강아지의 말투로 일기가 생성된다.

## Background

buddy-note는 사용자가 매일 사진과 짧은 메모를 남기면 강아지 1인칭 일기로 바꾸고, 시간이 지날수록 기록을 모아 보여주는 앱이다. 신규 사용자는 온보딩에서 강아지 정보를 입력했지만, 첫 홈 진입 시점에는 아직 화면 구조와 첫 행동을 모른다.

튜토리얼은 이 순간에 “어디에서 보고, 어디에서 모아보고, 어떻게 일기를 만드는지”를 아주 짧게 안내해야 한다. 기존의 기억 누적/공유 루프 설명은 제품 배경으로 남기되, 최초 진입 튜토리얼의 전면 메시지는 기능 이해와 첫 작성 유도에 집중한다.

## Goals

- 회원가입과 강아지 등록을 완료한 사용자에게 앱 최초 1회 진입 시 튜토리얼을 자동 노출한다.
- 튜토리얼은 같은 버전에서 완료 또는 건너뛰기 후 다시 자동 노출하지 않는다.
- 사용자가 탭하면 다음 단계로 넘어가는 3-step 흐름을 제공한다.
- 세 단계는 각각 캘린더 뷰, 주간 뷰, AI 일기 생성을 설명한다.
- 마지막 단계에서 `첫 일기 쓰기` CTA로 `/log`에 이동할 수 있게 한다.
- 홈 화면과 하단 네비게이션의 실제 구조와 튜토리얼 문구가 어긋나지 않게 한다.

## Non-Goals

- 결제, 멤버십, 무료 체험판 안내는 포함하지 않는다.
- 공개 프로필, 공유 이미지, 월간/연간 하이라이트는 이번 튜토리얼의 핵심 단계로 다루지 않는다.
- 기존 유저 전체에게 강제 재노출하지 않는다.
- 복잡한 coach mark, 화면 요소별 spotlight, guided tour positioning은 v1 범위에서 제외한다.
- analytics SDK 연동은 별도 작업으로 분리한다. 이 스펙은 상태 저장과 UX 동작만 잠근다.

## Target Moment

튜토리얼이 자동 노출되는 순간은 아래 흐름 직후다.

1. 사용자가 회원가입 또는 로그인을 완료한다.
2. 강아지 등록 온보딩을 완료한다.
3. 앱 홈(`/`)에 처음 진입한다.

이 시점의 사용자는 “이제 어디를 보면 되는지”, “하단 탭은 무엇인지”, “첫 일기는 어떻게 만드는지”를 알아야 한다.

## Trigger Rules

튜토리얼은 아래 조건을 모두 만족할 때 자동 노출한다.

- 로그인된 사용자다.
- 최소 1마리의 강아지 프로필이 있다.
- 현재 튜토리얼 버전을 완료하거나 건너뛴 기록이 없다.
- 사용자가 홈(`/`)에 도착했다.
- 세션 확인과 pet 조회가 끝났다.

첫 일기 존재 여부는 자동 노출을 막는 조건으로 사용하지 않는다. 사용자가 온보딩 직후 바로 예시 데이터나 이전 기록을 갖는 상태가 되더라도, 앱 구조 설명은 최초 1회 필요하기 때문이다.

## Suppression Rules

아래 상황에서는 자동 노출하지 않는다.

- 사용자가 같은 버전의 튜토리얼을 완료했다.
- 사용자가 같은 버전의 튜토리얼을 건너뛰었다.
- 반려동물 프로필이 없다.
- 로그인 상태가 아니거나 세션 확인이 끝나지 않았다.
- 현재 route가 홈(`/`)이 아니다.
- 오류 화면, 업로드 진행, 일기 생성 진행처럼 사용자의 현재 작업을 방해할 수 있는 상태다.

새 튜토리얼 버전은 핵심 기능 설명이나 노출 정책이 바뀌었을 때만 만든다. 단순 문구 수정이나 디자인 조정만으로는 기존 사용자에게 재노출하지 않는다.

## Tutorial Format

튜토리얼은 모바일 우선 bottom sheet로 구현한다. 데스크톱에서는 같은 컴포넌트를 중앙 modal처럼 보이게 할 수 있지만, 콘텐츠와 동작은 동일해야 한다.

형식 원칙:

- 총 3단계
- 사용자가 sheet 본문 또는 `다음` 버튼을 탭하면 다음 단계로 이동한다.
- 단계 indicator는 `1 / 3` 형태 또는 점 indicator로 표시한다.
- 언제든 닫기 가능하다.
- 닫기와 `나중에 볼게`는 dismissed로 저장한다.
- 마지막 단계에는 `첫 일기 쓰기` CTA를 제공한다.
- 배경 홈 화면과 하단 네비게이션바는 어둡게 가릴 수 있지만, 구조가 완전히 사라지지 않게 한다.

## Step Content

### Step 1: 캘린더 뷰

목적: 홈의 기본 화면이 날짜별 기록 확인 공간임을 알려준다.

Title:

> 캘린더에서 하루를 볼 수 있어

Body:

> 홈에서는 날짜마다 남긴 일기를 한눈에 볼 수 있어. 사진이 있는 날은 작은 썸네일로 표시돼.

Primary action:

- `다음`
- 탭하면 Step 2로 이동한다.

Tap behavior:

- sheet 본문을 탭해도 Step 2로 이동한다.

### Step 2: 주간 뷰

목적: 하단 네비게이션바 가장 좌측 기능이 `/week` 주간 뷰임을 알려준다.

Title:

> 왼쪽 탭에서 이번 주를 모아봐

Body:

> 하단 왼쪽 탭은 주간 뷰야. 최근 일기와 기억 포인트를 짧게 모아서 볼 수 있어.

Primary action:

- `다음`
- 탭하면 Step 3으로 이동한다.

Tap behavior:

- sheet 본문을 탭해도 Step 3으로 이동한다.

### Step 3: AI 일기 생성

목적: 첫 핵심 행동인 일기 작성을 유도한다.

Title:

> 사진을 남기면 일기가 써져

Body:

> 사진과 짧은 메모를 올리면 AI가 우리 강아지 말투로 오늘의 일기를 만들어줘. 첫 장부터 남겨볼까?

Primary action:

- `첫 일기 쓰기`
- 완료 상태를 저장하고 `/log`로 이동한다.

Secondary action:

- `나중에 볼게`
- dismissed 상태를 저장하고 홈에 머무른다.

Tap behavior:

- 마지막 단계에서 sheet 본문을 탭하는 것만으로는 `/log`로 이동하지 않는다. 명시적인 CTA를 눌러야 한다.

## Completion And Dismissal

- Step 3의 `첫 일기 쓰기`를 누르면 completed로 저장한다.
- Step 3까지 본 뒤 CTA를 누르지 않고 닫으면 dismissed로 저장한다.
- 어느 단계에서든 닫기 버튼을 누르면 dismissed로 저장한다.
- 같은 버전에서 completed와 dismissed는 모두 자동 재노출을 막는다.
- 사용자가 브라우저를 새로고침하거나 다른 기기에서 로그인해도 상태가 유지되어야 하므로 DB에 저장한다.
- localStorage나 sessionStorage는 보조 UX에만 사용할 수 있고, 자동 재노출 방지의 source of truth로 쓰지 않는다.

## Data Requirements

튜토리얼 상태는 사용자 단위로 저장한다.

필수 필드:

- `user_id`
- `tutorial_version`
- `completed_at`
- `dismissed_at`
- `created_at`
- `updated_at`

제약:

- `user_id`는 DB 컨벤션에 맞춰 반드시 이 이름을 사용한다.
- 동일 사용자와 동일 버전은 1행만 허용한다.
- 한 행에서 `completed_at`과 `dismissed_at`이 동시에 채워지지 않게 한다.
- RLS는 본인 row만 select/insert/update 가능해야 한다.

## UI Tone

- 한국어 기본
- 짧고 구체적인 문장
- 부드러운 반말/권유형
- 이모지 금지
- `스마트`, `혁신`, `올인원` 같은 범용 SaaS 표현 금지
- 색상은 `var(--color-*)` 토큰만 사용한다.
- CTA는 테라코타 계열 토큰을 사용한다.
- 과한 animation, gradient, pastel onboarding 표현은 사용하지 않는다.

## Accessibility

- sheet는 `role="dialog"`와 `aria-modal="true"`를 사용한다.
- 현재 단계 title을 `aria-labelledby`로 연결한다.
- 닫기 버튼에는 `aria-label="튜토리얼 닫기"`를 제공한다.
- keyboard 사용자는 `다음`, `첫 일기 쓰기`, `나중에 볼게`, 닫기 버튼을 tab으로 접근할 수 있어야 한다.
- Escape 키로 닫을 수 있으면 dismissed로 저장한다.

## Success Metrics

Primary metric:

- 튜토리얼 노출 후 24시간 내 첫 일기 생성률

Supporting metrics:

- Step 1 → Step 2 진행률
- Step 2 → Step 3 진행률
- Step 3 `첫 일기 쓰기` 클릭률
- 튜토리얼 dismissed 비율
- 튜토리얼 이후 `/week` 진입률
- 튜토리얼 이후 홈 캘린더 날짜 탭 비율

Guardrail metrics:

- 튜토리얼 노출 직후 홈 이탈률
- Step 1에서 즉시 닫기 비율
- `/log` 진입 후 일기 생성 중도 이탈률

## Risks

- 주간 뷰를 “모아보기”처럼 설명하면 `/logs` 화면과 혼동될 수 있다. 문구는 반드시 “주간 뷰”와 “하단 왼쪽 탭”으로 고정한다.
- 탭으로 넘기는 동작이 너무 숨겨지면 사용자가 멈출 수 있다. `다음` 버튼과 단계 indicator를 함께 둔다.
- 자동 노출 상태를 localStorage에만 저장하면 기기 변경 후 반복 노출된다. DB 저장을 기준으로 한다.
- 첫 진입 튜토리얼이 홈 화면을 과하게 가리면 앱 구조를 익히기 어렵다. bottom sheet 높이를 제한한다.
